About Me

Hi! I'm Katie, a front-end developer located in Berlin, Germany, specialising in static pages and react. I have a past as a mechanical engineer, and a passion for hacking, building, solving problems, travel, and learning languages... if you count computer and human languages, I've got 8 so far. I am currently working on deepening my knowledge of backend and dev-ops technologies, as well as exploring different tools in the front-end.

Work

Here you can find a few of my projects so far.

  • AFP-Alert

    Amanda Palmer is an American musician who tours a lot, and has a tendency to do free concerts here and there with short notice given on Twitter. I created a tool to watch her twitter and send out an email alert when she mentions a chosen town, so that people who sign up for the service will know if she is thinking of coming to their area. Tool and website built with meteor, database to hold collections of emails and keywords on Mongo, hosted on AWS. Available at: AFP-Alert.mcgnly.com

  • Basic Redux

    While I was teaching a couple of people development in react and how to incorporate redux, I built a tool to explain the basics and make it easy to interact with different elements to better understand how they fit together. Built using create-react-app, redux, less, and gulp.

  • relayr Browser SDK

    APIs make it possible for the average developer to interact with a given set of server-side functions and data, but SDKs make that job even easier, by abstracting away the tasks of making AJAX calls, interacting with OAuth, and other internal details. The browser SDK built for relayr makes it easy for a developer to use standard javascript to interact with their IOT devices and use the data for whatever projects they might have in mind. Built using ES6 and Webpack.

  • 99 Bottles

    Using the relayr browser SDK, I built a dashboard and small tool which, when paired with an IOT enabled scale, can report the number of bottles left in a case of beer. This concept can of course be extended to report any kind of weight-based counting task with recalibration of the scale. Developed as a demo for a large German grocery chain. Built using ES6, JQuery, and Webpack.

  • Amazon Alexa Cat-Facts

    As part of my interest in developing my skills in different directions, I learned how to build apps for the Amazon Alexa. These use voice-control to accomplish different tasks- in this case, to poll an API for a random fact about cats, and return the information to the user also over speech. Built using Node and Amazon Lambda. Available at: Amazon Alexa app store

  • Amazon Alexa Meat Thermometer

    The second in a series of apps built for Amazon Alexa, this app has a database of cooking temperatures for common cuts of meat, and interacts with the user over voice. I was tired of always having to look up the temperature for pork tenderlion... Built using Node and Amazon Lambda. Available at: Amazon Alexa app store

  • Amazon Alexa Space Station Locater

    Third in a series of apps built for Amazon Alexa. This one takes the current time and returns the point on Earth over which the International Space Station is at this very second. Built using Node and Amazon Lambda. Available at: Amazon Alexa app store

  • Million Monkeys

    An infinite number of monkeys on infinite typewriters given infinite time would eventually be able to produce the collected works of Shakespeare...As part of the process of learning to use Python, I developed a toy command-line app that, when given a particular string, will replicate it using randomly generated characters, and count the number of repetitions it takes to recreate the given word or phrase.

Notes on Design

This website was made with accessibility in mind.

  • Contrast: according to best practices set forth by WCAG AAA standard, the contrast between normal-sized text and background should be a minimum of 7:1. On this page it is 7.14:1

  • Colors: the palette here was kept to a relatively monochromatic scale, in order to be readable by people with different forms of color blindness, and the text is slightly darker than white in order to be more readable by people with dyslexia.

  • Screen readers: this page is tab-through-able, and on the first tab reveals a "skip to content" button, which bypasses the navbar.
